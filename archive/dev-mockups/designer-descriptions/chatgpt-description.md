A **desktop-style three-pane application shell** or **Obsidian or IDE-style workspace layout**. 

A clean design-language description would be:

> **A low-fidelity wireframe for a document-centric desktop app, using a three-pane resizable split-view layout with persistent application chrome. It has a left navigation/file-tree sidebar, a central editor/content workspace, a right inspector/properties panel, top global controls/toolbars, and a bottom status bar.**

More specific terms:

| Area                          | Designer term                                                                        |
| ----------------------------- | ------------------------------------------------------------------------------------ |
| Whole structure               | **Application shell**, **workspace layout**, **desktop app frame**                   |
| Left panel                    | **Navigation sidebar**, **file tree**, **project explorer**                          |
| Center panel                  | **Primary content area**, **editor canvas**, **workspace**, **document viewport**    |
| Right panel                   | **Inspector panel**, **properties panel**, **options sidebar**                       |
| Top rows                      | **Global header**, **title bar**, **toolbar**, **breadcrumb bar**, **mode switcher** |
| Bottom row                    | **Status bar**, **activity/status footer**, **metadata bar**                         |
| Resizable/collapsible regions | **Split panes**, **dockable panels**, **collapsible sidebars**                       |

The closest familiar patterns are:

* **IDE/editor layout**: like VS Code, Xcode, Sublime Text, or JetBrains apps.
* **Creative/pro authoring app layout**: like Figma, Photoshop, Final Cut, Notion-like editors with side panels.
* **Navigation-content-inspector pattern**: left side for structure, center for work, right side for contextual settings.

So the most concise label would be:

> **Three-pane editor interface with persistent app chrome and dockable side panels.**

Or, slightly more product-design-y:

> **A document-centric workspace shell with left navigation, central editing canvas, right inspector, top toolbars, and bottom status regions.**
